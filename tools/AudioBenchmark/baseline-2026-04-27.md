# Whisper Baseline — 2026-04-27

Decision-trace document for the v0.2.1 patch that remaps
`WhisperModelSize.Large` from `GgmlType.LargeV3` to `GgmlType.LargeV2`.

This document combines Phase 1 (matrix validation), Phase 1.5 (long-form
diagnosis + Whisper.net API audit), the model-variant comparison, and the
final patch decision. The raw measurements behind every claim live under
[`results/`](results/).

## Host environment

```
CUDA reported  : true   (driver-shipped nvcuda.dll present)
Vulkan reported: true   (driver-shipped vulkan-1.dll present)
System RAM     : 63 GB
Logical cores  : 28
WhisperEnvironment.RecommendModelSize(Accuracy) → Large
```

Single-host data. Lower-end hosts (CPU-only, < 16 GB RAM) will experience
different RTF cliffs and were not exercised in this run.

## Diagnosis — narrowing the cause

### Whisper.net 1.9.0 API audit

Reflected via `AudioBenchmark --probe-api`. Loop-mitigation knobs
available on `WhisperProcessorBuilder`:

```
WithNoContext()                    // condition_on_previous_text=false
WithTemperature(float)             // initial decoding temperature
WithTemperatureInc(float)          // fallback retry increment
WithLogProbThreshold(float)        // fallback trigger
WithEntropyThreshold(float)        // fallback trigger
WithSuppressRegex(string)          // mask repeated patterns
WithBeamSearchSamplingStrategy()   // alternate sampler with WithBeamSize / WithPatience
```

Both options A (`WithNoContext`) and the temperature fallback chain are
exposed by this Whisper.net version.

### A/B 1: VOA × Large × `WithNoContext`

CSV: same as VOA above (rows where `config = no_context`).

| config | WER | CER | TranscriptCh |
|---|---|---|---|
| default | 191.7 % | 155.6 % | 18,667 |
| no_context | 191.7 % | 155.6 % | 18,667 |

Output is byte-for-byte identical — same loop, same suffix.
**Hypothesis "condition_on_previous_text triggers the loop" is rejected.**

(Whisper.net Issue #171 separately notes that its `WithNoContext` may not
correspond exactly to whisper.cpp's `-mc 0` flag. Our measurement is
consistent with that — flipping this knob changes nothing on the looping
input.)

### A/B 2: VOA × Large × full temperature fallback

CSV: same VOA file (rows where `config = fallback_full`).
Settings: `WithTemperature(0.0).WithTemperatureInc(0.2).WithLogProbThreshold(-1.0).WithEntropyThreshold(2.4)`
(whisper.cpp standard).

Result: identical to default within rounding. Still loops.

The temperature-fallback retry path is therefore not engaging — either the
greedy decode never trips its trigger conditions on this particular
audio×model combination, or the retry path itself converges back into the
same beam. Either way, no fix.

### Hypothesis 3: Whisper Large-V3 weights specifically

Two community-confirmed observations point at the model itself:

- OpenAI whisper Issue #1606: Large-V3 hallucinates "Legenda Adriana
  Zanotto" every 30 seconds on 1 hour of pure silence; Large-V2 and
  Medium do not, on the same input.
- Whisper.net Issue #171: cite the same upstream behaviour.

Our tier-mapping data is consistent: Tiny / Base / Small / Medium are all
fine; only Large fails. `WhisperModelProvider.cs:120` mapped
`WhisperModelSize.Large → GgmlType.LargeV3`. Hypothesis: switch to
`GgmlType.LargeV2`.

## Variant comparison — V3 / V2 / Turbo

CSVs: [`results/2026-04-27-v2-validation.csv`](results/2026-04-27-v2-validation.csv),
[`results/2026-04-27-turbo-validation.csv`](results/2026-04-27-turbo-validation.csv).

Implementation: temporary `--ggml <type>` flag plus `InternalsVisibleTo`
on `FieldCure.DocumentParsers.Audio` so the benchmark can call
`WhisperModelProvider.GetModelPathByGgmlTypeAsync` (an internal helper)
without changing the public surface.

| Sample | Variant | WER | CER | RTF | Char | Loop? |
|---|---|---|---|---|---|---|
| KSS (15 × 5 s) | V3 (default) | 6.7 % | **0 %** | 0.136 | 305 | clean |
|  | **V2** | 9.7 % | 1.1 % | 0.133 | 307 | **clean** |
|  | Turbo | 13.9 % | 1.2 % | 0.109 | 302 | clean |
| VOA (16 min) | V3 | 191.7 % | 155.6 % | 0.060 | 18,667 | **LOOP** |
|  | **V2** | **2.2 %** | **2.1 %** | **0.022** | **8,202** | **clean** ✓ |
|  | Turbo | 20.4 % | 24 % | 0.011 | 9,829 | LOOP (different phrase) |
| BBC (40 min) | V3 | 313 % | 213 % | 0.080 | 64,614 | **LOOP** |
|  | **V2** | **2.6 %** | **1.6 %** | **0.024** | **21,533** | **clean** ✓ |
|  | Turbo | 9.7 % | 7.0 % | 0.012 | 22,498 | clean tail |

Spot-checks for the V2 entries show the regression on KSS is mostly
**surface-level conventions** (Korean spacing and Arabic-numeral vs Hangul-
numeral choices), not semantic errors:

```
GT:    마음 속으로 생각한 수에 2를 더하세요.
V3:    마음속으로 생각한 수에 2를 더하세요.
V2:    마음속으로 생각한 수에 '이'를 더하세요.    ← "2" → "이" (semantically equal)
```

Of the 4 KSS-V2 outliers contributing to the 1.1 % CER, only one is a real
character substitution (`낯은` → `나친`, single character). The rest are
spacing or numeric-notation differences.

Turbo (distilled from V3) inherits the looping behaviour: VOA loops,
BBC happens to terminate cleanly. Sample-dependent instability is not an
acceptable property for a default model.

## Decision

**v0.2.1: remap `WhisperModelSize.Large` → `GgmlType.LargeV2`.**

Rationale (data-driven):

| Criterion | V3 | V2 | Turbo |
|---|---|---|---|
| Long-form stability (VOA + BBC) | ❌ both loop | ✓ both clean | ⚠ VOA loops |
| Long-form quality | unmeasurable (loops) | best in matrix | best except looping |
| Korean clean-clip quality | best (CER 0 %) | acceptable (CER 1.1 %, ≈ surface) | similar to V2 |
| RTF on this host | 0.06–0.13 | 0.02–0.13 (often faster b/c no loop) | 0.01–0.11 |
| Memory | 531–1491 MB | 594–1305 MB | 387–1317 MB |

V2 wins on every long-form metric and is essentially indistinguishable
from V3 on Korean clean clips at the semantic level. The decision is a
single-line code change.

**Excluded options and why**:

- **Temperature fallback (option D)** — A/B showed it has no effect on
  this audio×model combination. Skipped.
- **VAD pre-chunking (option C)** — Defer to v0.3+. Not justified given
  V2 alone resolves the symptom for our user audience.
- **Turbo as new default** — Inherits V3's looping. Disqualified.
- **Cap default to Medium (option B)** — Now a worse choice than V2
  because V2 also delivers better quality on long-form than Medium.

## v0.2.1 patch

```diff
-    public static string GetModelFileName(WhisperModelSize modelSize) => modelSize switch
+    public static string GetModelFileName(WhisperModelSize modelSize) => modelSize switch
     {
         ...
-        WhisperModelSize.Large => "ggml-large-v3.bin",
+        WhisperModelSize.Large => "ggml-large-v2.bin",
         ...
     };

     private static GgmlType ToGgmlType(WhisperModelSize modelSize) => modelSize switch
     {
         ...
-        WhisperModelSize.Large => GgmlType.LargeV3,
+        WhisperModelSize.Large => GgmlType.LargeV2,
         ...
     };
```

Plus an XML doc comment at the mapping site referencing this document so
future maintainers can find the rationale without re-running the
benchmark.

### User-visible impact

- The previous `~/.fieldcure/whisper-models/ggml-large-v3.bin` cache
  becomes orphaned (≈ 3 GB). The new cache filename is
  `ggml-large-v2.bin`; first run after upgrade triggers a one-time
  download. Manual cleanup of the V3 file is recommended but not
  required (Whisper.net never reads it again).
- Memory baseline ~12 % higher (594 vs 531 MB on KSS) — irrelevant
  next to host RAM in practice.
- Korean transcripts may show different number-formatting and spacing
  conventions vs prior V3 output. WER comparisons against V3-era
  baselines need re-establishing.

## Regression verification (v0.2.1)

After applying the patch, the same VOA (16 min) and BBC (40 min) clips
were re-measured with default `WhisperModelSize.Large` (now backed by

The verification CSV is captured separately at
`results/2026-04-27-v021-regression.csv`; values must match the V2 rows
